Subqueries Randwissen AP2

Einstieg

Der Chef von TowerTech GmbH fragt: „Wer verdient mehr als der Durchschnitt?" Das Problem: Den Durchschnitt kennt niemand auswendig – er muss erst berechnet werden, bevor man ihn mit den einzelnen Gehältern vergleichen kann. Zwei getrennte Abfragen (erst Durchschnitt berechnen, dann Ergebnis von Hand einsetzen) wären umständlich. Eine Subquery (Unterabfrage) erledigt beides in einem einzigen SQL-Befehl: Sie ist eine SELECT-Abfrage innerhalb einer anderen Abfrage.

Konzept

Eine Subquery liefert einen Wert oder eine Werteliste, die direkt von der äußeren Abfrage weiterverwendet wird. Alle Beispiele beziehen sich auf die Tabellen abteilung und mitarbeiter.

Subquery in WHERE mit Vergleichsoperator

SELECT vollname, jahresgehalt FROM mitarbeiter WHERE jahresgehalt > (SELECT AVG(jahresgehalt) FROM mitarbeiter);

Die innere Abfrage (SELECT AVG(jahresgehalt) FROM mitarbeiter) liefert genau einen Wert (den Durchschnitt), die äußere Abfrage vergleicht jede Zeile damit.

Subquery mit IN

Liefert die innere Abfrage mehrere Werte, wird sie mit IN statt eines einzelnen Vergleichsoperators kombiniert:

SELECT vollname FROM mitarbeiter WHERE abteilung_id IN (SELECT abteilung_id FROM abteilung WHERE buero_standort = 'Essen');

Erst ermittelt die innere Abfrage alle Abteilungs-IDs mit Bürostandort Essen, dann filtert die äußere Abfrage alle Mitarbeitenden dieser Abteilungen.

Subquery im FROM (abgeleitete Tabelle)

Eine Subquery kann auch als eigene, temporäre Tabelle im FROM stehen – sie braucht dann einen Alias:

SELECT abt_avg.abteilung_id, abt_avg.durchschnitt FROM (SELECT abteilung_id, AVG(jahresgehalt) AS durchschnitt FROM mitarbeiter GROUP BY abteilung_id) AS abt_avg WHERE abt_avg.durchschnitt > 40000;

EXISTS

EXISTS prüft nur, ob die Subquery mindestens eine Zeile liefert – nicht, welche Werte sie enthält. Das ist oft effizienter, wenn nur die Existenz interessiert:

SELECT name FROM abteilung a WHERE EXISTS (SELECT 1 FROM mitarbeiter m WHERE m.abteilung_id = a.abteilung_id AND m.jahresgehalt > 50000);

Diese Abfrage findet alle Abteilungen, die mindestens eine Person mit einem Gehalt über 50000 € beschäftigen.

Praxisbeispiel

Der Chef von TowerTech GmbH stellt vier Fragen, die eine Auszubildende jeweils mit einer einzigen Abfrage im Editor oben beantworten soll:

  • „Wer verdient mehr als der Durchschnitt?" – Subquery in WHERE mit > und AVG().
  • „Wer arbeitet in einer Abteilung mit Sitz in Essen?" – Subquery mit IN.
  • „Welche Abteilungen liegen über einem Durchschnittsgehalt von 40000 €?" – Subquery im FROM als abgeleitete Tabelle, kombiniert mit GROUP BY.
  • „Welche Abteilungen haben mindestens eine Person mit über 50000 € Gehalt?" – EXISTS-Subquery.

Typische Prüfungsfrage: „Was ist der Unterschied zwischen einer Subquery mit IN und einer Subquery mit EXISTS?" – Antwort: IN vergleicht einen Wert mit einer Liste von Werten aus der Subquery; EXISTS prüft nur, ob die Subquery überhaupt eine Zeile liefert, ohne deren konkrete Werte zu betrachten.

Merksatz

Eine Subquery ist eine SELECT-Abfrage innerhalb einer anderen Abfrage. In WHERE mit Vergleichsoperator für einen Einzelwert (z. B. AVG()), mit IN für eine Werteliste, im FROM als abgeleitete Tabelle mit Alias, und mit EXISTS um nur zu prüfen, ob überhaupt eine passende Zeile existiert.
