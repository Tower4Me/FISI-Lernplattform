IPv6-Grundlagen Kern AP1 AP2

Einstieg

Stell dir vor, eine kleine Stadt hätte irgendwann so viele Häuser gebaut, dass die vierstelligen Hausnummern nicht mehr ausreichen — jede mögliche Kombination ist längst vergeben. Die Lösung: ein neues, viel größeres Nummernsystem, das niemals ausgeht. Genau das ist die Motivation hinter IPv6: IPv4 hat nur rund 4,3 Milliarden Adressen, und die sind angesichts von Milliarden vernetzter Geräte weltweit längst knapp. IPv6 stellt eine astronomisch größere Anzahl an Adressen bereit — und bringt dabei gleich neue Mechanismen mit, mit denen sich Geräte ihre Adresse selbst „aussuchen“ können, ganz ohne zentrale Verwaltung.

Konzept

Eine IPv6-Adresse ist 128 Bit lang (gegenüber 32 Bit bei IPv4) und wird in acht Blöcken zu je 16 Bit hexadezimal geschrieben, getrennt durch Doppelpunkte, z. B. 2001:0db8:0000:0000:0000:ff00:0042:8329. Zur Vereinfachung dürfen führende Nullen in einem Block weggelassen und eine zusammenhängende Folge von Null-Blöcken einmal pro Adresse durch :: ersetzt werden — verkürzt also 2001:db8::ff00:42:8329.

Adresstypen

Link-Localfe80::/10APIPA (169.254.0.0/16)nur im lokalen Netzsegment, wird jedem Interface automatisch zugewiesen

Eine Link-Local-Adresse erhält jede IPv6-fähige Netzwerkschnittstelle automatisch, sobald sie aktiviert wird — sie ist Voraussetzung für grundlegende Netzwerkprotokolle (z. B. Nachbarschaftserkennung) und wird nie über den Router hinaus weitergeleitet.

Automatische Adressvergabe: SLAAC

SLAAC (Stateless Address Autoconfiguration) erlaubt es einem Gerät, sich seine eigene Global-Unicast-Adresse ganz ohne DHCP-Server selbst zu erzeugen:

  1. Der Router sendet regelmäßig Router Advertisements (RA) mit dem Netzwerkpräfix (z. B. 2001:db8:abcd:1::/64) ins lokale Netz.
  2. Das Gerät übernimmt dieses Präfix als Netzanteil seiner Adresse.
  3. Den Hostanteil (Interface-Identifier) erzeugt das Gerät selbst — häufig per EUI-64-Verfahren.

EUI-64: Interface-Identifier aus der MAC-Adresse

Beim EUI-64-Verfahren wird die 48 Bit lange MAC-Adresse der Netzwerkkarte zu einem 64 Bit langen Interface-Identifier erweitert:

  • Die MAC-Adresse wird in der Mitte aufgetrennt (24 Bit Hersteller-Kennung + 24 Bit Geräte-Kennung).
  • Dazwischen wird die feste Bitfolge FFFE eingefügt — aus 48 Bit werden so 64 Bit.
  • Ein einzelnes Bit (das „Universal/Local“-Bit) wird invertiert, um anzuzeigen, dass die Adresse global eindeutig abgeleitet wurde.

Damit lässt sich aus jeder MAC-Adresse eindeutig ein Interface-Identifier ableiten, ohne dass ein Server Adressen verwalten oder Kollisionen verhindern muss.

Praxisbeispiel

Ein Notebook wird in ein Firmennetz eingebunden, das sowohl IPv4 als auch IPv6 (Dual-Stack) betreibt.

  • Sobald die Netzwerkkarte aktiv ist, erhält sie sofort eine Link-Local-Adresse (fe80::…) — ganz ohne Router oder DHCP, ausschließlich für Kommunikation im lokalen Segment.
  • Der Router im Netz sendet ein Router Advertisement mit dem Präfix 2001:db8:abcd:5::/64.
  • Das Notebook führt SLAAC aus: Es übernimmt das Präfix und bildet per EUI-64 aus seiner MAC-Adresse einen eindeutigen Interface-Identifier — ohne dass ein DHCPv6-Server befragt werden musste.
  • Die IT-Administration erkennt am Aufbau der Adresse (Präfix 2001:db8:abcd:5::/64) sofort, aus welchem Subnetz das Gerät stammt — nützlich bei der Fehlersuche im Netzwerklog.

Typische Prüfungsfrage: „Welchen IPv4-Mechanismus ersetzt die IPv6-Link-Local-Adresse funktional?“ – Antwort: APIPA (169.254.0.0/16) — beide vergeben automatisch eine Segment-lokale Adresse, wenn kein DHCP-Server verfügbar ist bzw. grundsätzlich beim Interface-Start.

Merksatz

IPv6 = 128 Bit, hexadezimal in 8 Blöcken, :: kürzt Nullfolgen einmal pro Adresse. Global Unicast = öffentlich, Unique Local = privat, Link-Local (fe80::/10) = automatisch pro Interface, nur lokal. SLAAC lässt Geräte sich ohne DHCP selbst adressieren, EUI-64 leitet den Interface-Identifier aus der MAC-Adresse ab.
