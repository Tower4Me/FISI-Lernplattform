URL, URI & URN Kern AP1

Einstieg

Eine URI ist wie der Oberbegriff „Adressierung von irgendetwas“. Eine URL ist eine vollständige Wegbeschreibung: „Geh zu dieser Straße, dieser Hausnummer, über diesen Weg“ — sie sagt, wo etwas liegt und wie man hinkommt. Eine URN ist dagegen wie eine Sozialversicherungsnummer: Sie identifiziert eindeutig, was etwas ist, egal wo es sich gerade befindet — ein Buch bleibt über seine ISBN identifizierbar, egal in welchem Regal welcher Buchhandlung es gerade steht.

Konzept

URI, URL und URN im Verhältnis

BegriffBedeutung
URI (Uniform Resource Identifier) Oberbegriff — identifiziert eine Ressource eindeutig. URL und URN sind beides spezielle Arten von URIs.
URL (Uniform Resource Locator) gibt zusätzlich an, wo eine Ressource zu finden ist und wie man darauf zugreift (Protokoll + Adresse + Pfad) — die im Alltag mit Abstand häufigste Form einer URI
URN (Uniform Resource Name) identifiziert eine Ressource nur über einen eindeutigen Namen, unabhängig vom Ort — z. B. urn:isbn:0451450523 für ein bestimmtes Buch, egal wo es aktuell erhältlich ist

Aufbau einer URL

BestandteilBeispielBedeutung
Schema/Protokollhttps://welches Protokoll für den Zugriff genutzt wird
Subdomainshop.optionaler Teilbereich der Domain
Domaintechnord.deeigentliche Adresse des Servers
Port:8443optional — welcher Netzwerk-Port angesprochen wird (Standard bei HTTPS: 443)
Pfad/produkte/listewelche Ressource auf dem Server angefragt wird
Query-String?kategorie=notebooks&sortierung=preiszusätzliche Parameter als Schlüssel-Wert-Paare
Fragment#seite2Sprungmarke innerhalb der geladenen Seite, wird nicht an den Server gesendet

Vollständig zusammengesetzt ergibt das: https://shop.technord.de:8443/produkte/liste?kategorie=notebooks&sortierung=preis#seite2

Praxisbeispiel

Ein Kunde meldet dem Support der TechNord GmbH, dass eine bestimmte Shop-URL im Firmennetz nicht erreichbar ist: https://shop.technord.de:8443/produkte/liste?kategorie=notebooks&sortierung=preis#seite2

  • Schema: https — verschlüsselter Zugriff
  • Domain: shop.technord.de
  • Port: 8443 — ungewöhnlich, da der HTTPS-Standardport eigentlich 443 ist
  • Pfad: /produkte/liste
  • Query-String: Filterung nach Kategorie und Sortierung
  • Fragment: #seite2 — Sprung zu Seite 2 der Produktliste innerhalb der bereits geladenen Seite

Der Techniker erkennt: Der ungewöhnliche Port 8443 statt des Standardports 443 ist die Ursache — die Firmen-Firewall blockiert ausgehende Verbindungen auf diesem nicht-freigegebenen Port. Nach Freigabe des Ports funktioniert der Zugriff.

Merksatz

URI ist der Oberbegriff, URL sagt „wo und wie“ (Protokoll, Domain, Pfad), URN sagt nur „was“ (eindeutiger Name, ortsunabhängig). Eine URL besteht aus Schema, optionaler Subdomain, Domain, optionalem Port, Pfad, Query-String und Fragment — das Fragment wird nie an den Server übertragen.
