Deployment-Formate (ISO, WIM) Randwissen AP1

Einstieg

Eine ISO-Datei ist wie eine Fotokopie eines ganzen Buchs samt Einband und Bindung — Byte für Byte, inklusive der kompletten Struktur des Original-Datenträgers. Eine WIM-Datei ist dagegen eher wie eine flexible E-Book-Ausgabe desselben Inhalts: Sie lässt sich auf beliebiger Hardware neu „aufschlagen“ und kann sogar mehrere Sprachausgaben (Windows-Editionen) in einer einzigen Datei enthalten, aus denen man bei der Installation wählt.

Konzept

ISO — das Abbild eines Datenträgers

Eine ISO-Datei (benannt nach dem Standard ISO 9660 für optische Datenträger) ist ein exaktes Abbild (Image) eines gesamten Datenträgers oder Dateisystems, Byte für Byte — inklusive der kompletten Struktur des Datenträgers, nicht nur der enthaltenen Dateien.

EinsatzBeschreibung
Betriebssystem-InstallationWindows, Linux, macOS werden als ISO verteilt — herunterladen, auf USB-Stick schreiben (z. B. mit Rufus oder dd), davon booten
Virtuelle MaschinenVMware, VirtualBox und Hyper-V können ISO-Dateien direkt als virtuelles DVD-Laufwerk einbinden, ohne physischen Datenträger
PXE-Bootbeim Netzwerk-Deployment über PXE (Preboot Execution Environment) wird oft eine ISO bzw. deren Inhalt über das Netzwerk bereitgestellt

Unter Linux: dd if=windows11.iso of=/dev/sdb bs=4M status=progress schreibt die ISO auf einen USB-Stick (Achtung: überschreibt das Zielgerät vollständig) — mount -o loop windows11.iso /mnt/iso hängt die ISO dagegen nur ein, um den Inhalt anzusehen, ohne sie zu entpacken.

WIM — Windows Imaging Format

WIM ist das von Microsoft entwickelte Format für Betriebssystem-Images, speziell für das Deployment von Windows entwickelt. Der entscheidende Unterschied zu ISO: Eine ISO ist ein Abbild eines Datenträgers, WIM ist ein Abbild eines Dateisystems — dadurch hardware-unabhängig, komprimiert, und mehrere Windows-Editionen (Home, Pro, Enterprise) können in einer einzigen WIM-Datei stecken; die gewünschte Edition wird erst bei der Installation ausgewählt.

DateiBedeutung
install.wim / install.esddas eigentliche, komprimierte Windows-Dateisystem auf jedem Installationsmedium (sources\install.wim) — die .esd-Variante ist stärker komprimiert
boot.wimdas Boot-Image mit Windows PE (Preinstallation Environment) — ein minimales Windows, das nur für die Installation gestartet wird

Verwaltet wird WIM über das Tool DISM (Deployment Image Servicing and Management): z. B. dism /Get-WimInfo /WimFile:install.wim zeigt, welche Editionen enthalten sind, dism /Apply-Image … packt das Image auf ein Ziellaufwerk aus.

Beim PXE-Boot lädt ein Client zunächst über das Netzwerk boot.wim (Windows PE) bzw. ein Linux-Boot-Image, um danach über Windows Deployment Services (WDS) oder einen anderen Deployment-Server das eigentliche install.wim zu empfangen und zu installieren — alle beschriebenen Formate arbeiten hier zusammen.

Praxisbeispiel

Die TechNord GmbH rollt Windows auf 50 neuen Arbeitsplatzrechnern aus:

  • Einzelinstallation zum Testen: Ein Techniker lädt die Windows-ISO herunter und bindet sie direkt in eine virtuelle Testmaschine ein, ohne einen physischen Datenträger zu benötigen.
  • Massenverteilung im Netzwerk: Für die restlichen 49 Rechner nutzt die IT-Abteilung PXE-Boot: Jeder Client lädt zunächst boot.wim (Windows PE) über das Netzwerk, danach überträgt der Windows-Deployment-Server das eigentliche install.wim.
  • Edition wählen: Mit dism /Get-WimInfo /WimFile:install.wim prüft der Techniker vorab, welche Windows-Editionen im WIM-Image enthalten sind, und wählt „Pro“ für die Installation aus — eine einzige WIM-Datei genügt für alle benötigten Editionen.

Merksatz

ISO = exaktes Abbild eines gesamten Datenträgers (ISO 9660). WIM = Abbild eines Dateisystems, hardware-unabhängig, komprimiert, mehrere Editionen in einer Datei — verwaltet über DISM. install.wim = eigentliches System, boot.wim = Windows-PE-Boot-Image. PXE-Boot kombiniert beide für die Netzwerkinstallation.
