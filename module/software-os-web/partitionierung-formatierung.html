Partitionierung & Formatierung Kern AP1 AP2

Einstieg

Ein neuer, unbenutzter Datenträger ist wie ein leeres Grundstück. Partitionierung zieht darauf zunächst rechtlich getrennte Grundstücksgrenzen (Partitionen) — aus einem großen Grundstück können mehrere unabhängige Parzellen werden. Formatierung errichtet anschließend auf einer solchen Parzelle ein konkretes Gebäude (ein Dateisystem wie NTFS oder ext4) — erst dann lässt sich die Parzelle tatsächlich nutzen.

Konzept

Partitionierung

Partitionierung teilt einen physischen Datenträger in logisch getrennte Bereiche (Partitionen) auf. Jede Partition kann ein eigenes Dateisystem und einen eigenen Laufwerksbuchstaben (Windows) bzw. Mountpoint (Linux) erhalten (siehe Einheit Linux-Konsole: /dev/sda = gesamter Datenträger, /dev/sda1 = erste Partition darauf).

Partitionstabellen: MBR vs. GPT

MBR (Master Boot Record)GPT (GUID Partition Table)
Alter/Verbreitung älterer Standard moderner Standard
Partitionsanzahl maximal 4 primäre Partitionen (oder 3 primäre + 1 erweiterte mit mehreren logischen Partitionen darin) praktisch unbegrenzt viele Partitionen
Maximale Datenträgergröße bis 2 TiB nutzbar unterstützt Datenträger deutlich über 2 TiB
Boot-Voraussetzung klassisches BIOS-Booten benötigt in der Regel UEFI (siehe Einheit BIOS/UEFI & Bootvorgang)

Ein Datenträger größer als 2 TiB muss mit GPT partitioniert werden — mit MBR wäre nur ein Teil der Kapazität überhaupt nutzbar.

Formatierung

Formatierung legt ein konkretes Dateisystem (z. B. NTFS, ext4, FAT32) auf einer Partition an — erst danach lässt sich die Partition tatsächlich zum Speichern von Dateien nutzen.

FormatierungsartVorgang
Schnellformatierung löscht nur den Dateisystem-Index (vergleichbar mit dem Löschverhalten von Inode/MFT, siehe Einheit Dateisysteme-Grundlagen) — die eigentlichen Datenbytes bleiben zunächst physisch erhalten, bis sie überschrieben werden; schnell, aber ohne Prüfung des Datenträgers
Vollständige Formatierung prüft zusätzlich den gesamten Datenträger auf defekte Sektoren (Bad Blocks) — deutlich langsamer, aber sinnvoll bei neuen oder verdächtigen Datenträgern

Werkzeuge unter Windows

  • Datenträgerverwaltung — grafisches Werkzeug zum Erstellen, Löschen und Formatieren von Partitionen
  • Diskpart — Kommandozeilenwerkzeug für dieselben Aufgaben, geeignet für Automatisierung/Skripte

Praxisbeispiel

Ein Techniker der TechNord GmbH baut eine neue 4-TB-Festplatte in einen Server ein:

  • Partitionstabelle wählen: Da die Platte mit 4 TB deutlich über der 2-TiB-Grenze von MBR liegt, wird sie mit GPT partitioniert — sonst wäre nur ein Bruchteil der Kapazität nutzbar.
  • Formatierung: Da es sich um eine neue, bisher ungetestete Platte handelt, wählt der Techniker eine vollständige Formatierung mit NTFS, um gleichzeitig auf defekte Sektoren zu prüfen.
  • Vergleichsfall: Bei einer bereits geprüften, aber neu zu belegenden internen SSD reicht dagegen eine schnelle Schnellformatierung, da keine erneute vollständige Sektorprüfung nötig ist.

Merksatz

Partitionierung teilt einen Datenträger logisch auf, Formatierung legt darauf ein Dateisystem an. MBR: max. 4 primäre Partitionen, bis 2 TiB, BIOS-Boot. GPT: praktisch unbegrenzt viele Partitionen, über 2 TiB, benötigt meist UEFI. Schnellformatierung löscht nur den Index, vollständige Formatierung prüft zusätzlich auf defekte Sektoren.
