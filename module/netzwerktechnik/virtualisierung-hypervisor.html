Virtualisierung & Hypervisor Kern AP1 AP2

Einstieg

Ein großes Mehrfamilienhaus lässt sich in einzelne, komplett voneinander abgeschlossene Wohnungen aufteilen — jede Partei hat ihre eigene Küche, ihr eigenes Bad, ihren eigenen abschließbaren Eingang, obwohl alle dasselbe Gebäude, dieselben Leitungen und dasselbe Fundament nutzen. Ein Mieter merkt im Alltag nichts davon, dass nebenan weitere Wohnungen im selben Haus existieren. Genau so funktioniert Virtualisierung: Ein einzelner physischer Server wird in mehrere logisch komplett getrennte „virtuelle Wohnungen“ (virtuelle Maschinen) aufgeteilt, die sich dieselbe Hardware teilen, ohne sich gegenseitig zu beeinflussen.

Konzept

Virtualisierung erzeugt aus einer physischen Ressource mehrere logisch unabhängige, virtuelle Instanzen. Die Steuerungssoftware, die diese Aufteilung übernimmt, heißt Hypervisor.

Hypervisor-Typen

TypAufbauBeispiele
Typ 1 (Bare-Metal)läuft direkt auf der Hardware, ohne zugrunde liegendes Host-Betriebssystem — die VMs greifen direkt über den Hypervisor auf die Hardware zuVMware ESXi, Microsoft Hyper-V (Server-Rolle), Proxmox VE
Typ 2 (Hosted)läuft als Anwendung innerhalb eines normalen Host-Betriebssystems — ein zusätzlicher Softwarelayer zwischen Hardware und VMVMware Workstation, VirtualBox

Typ-1-Hypervisoren sind performanter und werden für den produktiven Serverbetrieb im Rechenzentrum eingesetzt, da keine zusätzliche Betriebssystemschicht Ressourcen verbraucht. Typ-2- Hypervisoren sind einfacher einzurichten und eignen sich gut für Testumgebungen auf einem gewöhnlichen Arbeitsplatzrechner.

Vorteile der Virtualisierung

  • Bessere Hardwareauslastung — mehrere VMs teilen sich die Ressourcen eines physischen Servers, statt dass jede Anwendung einen eigenen, oft kaum ausgelasteten Server benötigt.
  • Isolation — ein Absturz oder Sicherheitsvorfall in einer VM betrifft grundsätzlich nicht die anderen VMs auf derselben Hardware.
  • Flexibilität — neue VMs lassen sich innerhalb von Minuten bereitstellen, klonen oder auf andere Hardware verschieben (Live-Migration).
  • Snapshot/Rollback — der Zustand einer VM lässt sich vor riskanten Änderungen sichern und bei Bedarf zurücksetzen.

Praxisbeispiel

Ein IT-Systemhaus modernisiert die Serverlandschaft eines Kunden, der bisher fünf einzelne, jeweils schwach ausgelastete Server-Hardware-Boxen betreibt (Fileserver, Mailserver, Anwendungsserver, Datenbankserver, Druckserver).

  • Alle fünf Serverrollen werden als virtuelle Maschinen auf einer einzigen leistungsstarken physischen Servermaschine mit Typ-1-Hypervisor (z. B. VMware ESXi) konsolidiert.
  • Vor einem geplanten Update des Mailservers erstellt der Administrator einen Snapshot — schlägt das Update fehl, wird die VM in Sekunden auf den vorherigen Zustand zurückgesetzt.
  • Für Testzwecke installiert ein Entwickler zusätzlich VirtualBox (Typ-2-Hypervisor) auf seinem eigenen Arbeitsplatzrechner, um eine neue Softwareversion isoliert zu testen, ohne das Produktivsystem zu gefährden.
  • Durch die Konsolidierung sinken Stromverbrauch, Kühlbedarf und Wartungsaufwand deutlich gegenüber fünf einzelnen physischen Servern.

Typische Prüfungsfrage: „Warum wird für den produktiven Servereinsatz ein Typ-1-Hypervisor gegenüber einem Typ-2-Hypervisor bevorzugt?“ – Antwort: Ein Typ-1-Hypervisor läuft direkt auf der Hardware ohne zusätzliches Host-Betriebssystem und bietet dadurch bessere Performance und geringeren Ressourcen-Overhead als ein Typ-2-Hypervisor, der innerhalb eines normalen Betriebssystems läuft.

Merksatz

Virtualisierung teilt physische Ressourcen in mehrere logisch getrennte virtuelle Instanzen (Server, Desktop oder Anwendung). Typ-1-Hypervisor läuft direkt auf der Hardware (Bare-Metal, produktiv), Typ-2-Hypervisor läuft innerhalb eines Host-Betriebssystems (einfacher, für Tests). Vorteile: bessere Auslastung, Isolation, Flexibilität, Snapshots.
