Windows-Konsole & ACL Kern AP1 AP2

Einstieg

Das einfache Linux-Rechtesystem ist wie ein Festival mit drei Armbandfarben: eine Farbe für Personal, eine für Presse, eine für alle übrigen Gäste — grob, aber schnell verständlich. Windows arbeitet dagegen wie ein exklusives Event mit einem Türsteher, der eine detaillierte Gästeliste (ACL) prüft: Jede einzelne Person steht namentlich mit genau festgelegten Rechten darauf — manche dürfen in die Küche, manche nur in die Lounge, manche stehen sogar explizit auf der Verbotsliste, obwohl sie eigentlich zu einer erlaubten Gruppe gehören.

Konzept

Was ist eine ACL?

Windows nutzt ein deutlich detaillierteres Berechtigungssystem als das Linux-rwx-Modell: die ACL (Access Control List, Zugriffssteuerungsliste). Statt drei Gruppen mit je drei Rechten enthält eine ACL beliebig viele ACEs (Access Control Entries) — jeder Eintrag legt für einen einzelnen Benutzer oder eine Gruppe fest, welche Aktionen erlaubt oder verboten sind.

RechtBedeutung
ReadDateiinhalt und Attribute lesen
WriteInhalt ändern, Attribute setzen
ExecuteProgramm oder Skript starten
DeleteDatei oder Ordner löschen
Full Controlalle Rechte inklusive Berechtigungen selbst ändern
Denyexplizites Verbot — überschreibt ein an anderer Stelle gewährtes Recht

Windows-Berechtigungen werden standardmäßig vererbt: Dateien in einem Ordner übernehmen automatisch die Berechtigungen des übergeordneten Ordners. Das vereinfacht die Verwaltung, kann aber zu unerwartetem Verhalten führen, wenn an einzelnen Dateien zusätzlich explizite Rechte gesetzt werden.

ACL über die Kommandozeile: icacls

Der moderne Windows-Befehl zur Verwaltung von ACLs ist icacls (der ältere Befehl cacls gilt als veraltet):

BefehlWirkung
icacls datei.txtzeigt die aktuellen Berechtigungen der Datei an
icacls datei.txt /grant Ben:(R)gewährt dem Benutzer „Ben“ Leserecht
icacls datei.txt /remove Benentfernt sämtliche Berechtigungseinträge für „Ben“
icacls datei.txt /grant Administratoren:(F)gewährt der Gruppe „Administratoren“ Vollzugriff (F = Full Control)

Praxisbeispiel

Ein Techniker der TechNord GmbH pflegt einen Windows-Fileserver:

  • Rechte prüfen: icacls finanzen\ zeigt, welche Benutzer und Gruppen aktuell Zugriff auf den Ordner „Finanzen“ haben.
  • Ausgeschiedener Mitarbeiter: Mit icacls finanzen\ /remove MMustermann wird dessen Zugriffsberechtigung vollständig entfernt.
  • Explizites Verbot: Eine bestimmte Abteilung soll trotz allgemeiner Lesefreigabe keinen Zugriff auf einen Unterordner mit besonders sensiblen Finanzdaten haben — ein Deny-Eintrag setzt sich gegenüber der allgemeinen Erlaubnis durch.
  • Netzwerkdiagnose: ipconfig /all zeigt die aktuelle IP-Konfiguration des Servers, getmac liefert die MAC-Adresse für eine feste DHCP-Reservierung.

Merksatz

ACL = Liste von ACEs, viel feiner als Linux-rwx, mit Vererbung und explizitem Deny (überschreibt Allow). Moderner Befehl: icacls (nicht cacls, veraltet). dir entspricht Linux ls, ipconfig zeigt die Netzwerkkonfiguration, getmac die MAC-Adresse des Netzwerkadapters.
