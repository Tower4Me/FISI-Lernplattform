Anwendungsprotokolle AP1 AP2

Einstieg

In einem Bürogebäude gibt es für jede Art von Anliegen einen eigenen Schalter mit eigenen Spielregeln: Am Postschalter gilt ein anderes Ablaufformular als am Bewerbungsschalter, und beim Sicherheitsdienst am Empfang wieder ein anderes. Wer ein Paket abholen will, geht zum Postschalter — nicht zum Empfang. Genauso haben sich im Internet für unterschiedliche Aufgaben (Webseiten abrufen, E-Mails holen, Dateien übertragen, sich per Fernzugriff einloggen) jeweils eigene Anwendungsprotokolle etabliert — feste Regelwerke, nach denen zwei Programme miteinander „sprechen“.

Konzept

Anwendungsprotokolle arbeiten auf OSI-Schicht 7 und definieren das Format und den Ablauf der Kommunikation zwischen Client und Server für einen bestimmten Zweck. Jedes Protokoll ist fest mit einem oder mehreren Standard-Ports verknüpft.

ProtokollPortZweck
HTTP80 (TCP)Abruf von Webseiten, unverschlüsselt
HTTPS443 (TCP)Abruf von Webseiten, TLS-verschlüsselt
FTP20/21 (TCP)Dateiübertragung (Steuerkanal 21, Datenkanal 20)
SMTP25 / 587 (TCP)Versand von E-Mails zwischen Mailservern bzw. vom Client zum Server
IMAP143 / 993 (TCP)Abruf von E-Mails, Nachrichten bleiben synchronisiert auf dem Server
POP3110 / 995 (TCP)Abruf von E-Mails, klassischerweise Herunterladen und Löschen vom Server
SSH22 (TCP)Verschlüsselter Fernzugriff auf die Kommandozeile eines Systems
RDP3389 (TCP)Fernzugriff auf eine grafische Windows-Desktopoberfläche
SNMP161/162 (UDP)Überwachung und Verwaltung von Netzwerkgeräten (z. B. Switches, Router)
NTP123 (UDP)Synchronisation der Systemzeit zwischen Geräten
LDAP389 / 636 (TCP)Abfrage und Verwaltung von Verzeichnisdiensten (z. B. Active Directory), 636 = LDAPS (verschlüsselt)

Eine wichtige Unterscheidung betrifft IMAP vs. POP3: IMAP hält den Posteingang auf dem Server als „Wahrheit“ und synchronisiert ihn auf mehrere Geräte (E-Mails bleiben nach dem Abruf auf dem Server erhalten) — praktisch, wenn dieselbe Mailbox von mehreren Geräten aus genutzt wird. POP3 lädt E-Mails klassischerweise herunter und entfernt sie vom Server — gedacht für ein einzelnes Gerät als alleinigen Zugriffspunkt.

Praxisbeispiel

Ein Systemadministrator richtet den Arbeitsplatz einer neuen Mitarbeiterin ein.

  • Der E-Mail-Client wird per IMAP (Port 993, verschlüsselt) konfiguriert, damit dieselbe Mailbox synchron auf Notebook und Smartphone verfügbar ist.
  • Für den Fernzugriff auf den Arbeitsplatz-PC vom Homeoffice aus wird RDP (Port 3389) über eine gesicherte VPN-Verbindung freigeschaltet.
  • Die IT-Abteilung wartet den Server im Rechenzentrum ausschließlich über SSH (Port 22) — verschlüsselter Kommandozeilenzugriff statt unsicherer Klartext-Protokolle.
  • Die Anmeldedaten der Mitarbeiterin werden zentral über LDAP gegen den Verzeichnisdienst der Firma (Active Directory) geprüft.
  • Alle Server im Rechenzentrum synchronisieren ihre Systemzeit automatisch per NTP — wichtig, damit Protokolldateien (Logs) zeitlich korrekt zugeordnet werden können.

Typische Prüfungsfrage: „Welches Protokoll wird für den verschlüsselten Fernzugriff auf die Kommandozeile eines Linux-Servers verwendet, und über welchen Port?“ – Antwort: SSH über Port 22.

Merksatz

Jedes Anwendungsprotokoll hat einen festen Standard-Port: HTTP 80 / HTTPS 443, FTP 20/21, SMTP 25/587, IMAP 143/993, POP3 110/995, SSH 22, RDP 3389, SNMP 161/162, NTP 123, LDAP 389/636. IMAP synchronisiert E-Mails auf dem Server, POP3 lädt sie klassisch herunter und löscht sie dort.
