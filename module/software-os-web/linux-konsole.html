Linux-Konsole & Dateiberechtigungen AP1 AP2

Einstieg

Windows organisiert Laufwerke wie separate Aktenschränke (C:, D:, E:) nebeneinander. Linux kennt nur einen einzigen, riesigen Aktenschrank mit einer festen Schublade an der Wurzel — jeder weitere Datenträger wird als zusätzliche Schublade an einer bestimmten Stelle in diesen einen Schrank eingeschoben (Mounting). Und wer welche Schublade öffnen, hineinschauen oder etwas hineinlegen darf, regelt ein Dreier-Schlüsselsystem: ein Schlüssel für den Eigentümer, ein gemeinsamer Team-Schlüssel für die Gruppe, und ein allgemeiner Schlüssel für alle anderen.

Konzept

Verzeichnisbaum und Mounting

Linux kennt keine Laufwerksbuchstaben wie C: oder D:. Stattdessen gibt es einen einzigen Verzeichnisbaum, beginnend bei der Wurzel /. Jeder weitere Datenträger wird einem Verzeichnis in diesem Baum zugewiesen — diesen Vorgang nennt man Mounting (Einhängen), das Zielverzeichnis heißt Mountpoint (z. B. /mnt/usb). Erst ab dem Moment des Mountens ist der Inhalt des Datenträgers über diesen Pfad erreichbar; nach dem Aushängen (umount) bleibt der Mountpoint selbst bestehen, nur sein Inhalt ist wieder unerreichbar.

Die Datei /etc/fstab (File System Table) steuert, welche Dateisysteme beim Systemstart automatisch gemountet werden:

GerätMountpointTypOptionen
/dev/sda1/ext4defaults
UUID=1A2B-3C4D/mnt/usbvfatauto,user

Eine fehlerhafte /etc/fstab kann dazu führen, dass Linux beim Booten hängenbleibt — deshalb vor jeder Änderung ein Backup anlegen und mit mount -a testen (versucht alle fstab-Einträge zu mounten, ohne neu zu starten).

Das rwx-Rechtesystem

Jede Datei und jedes Verzeichnis hat drei Rechte für drei Gruppen — kurz „ugw“ genannt:

Rechtbei Dateienbei Verzeichnissen
r (read)Inhalt lesenInhalt auflisten (ls)
w (write)Inhalt ändernDateien erstellen/löschen
x (execute)als Programm ausführenVerzeichnis betreten (cd)
Gruppe (u/g/w)Bedeutung
u — User/Ownerder Eigentümer der Datei (Ersteller wird automatisch Eigentümer)
g — Groupeine Benutzergruppe, alle Mitglieder teilen sich diese Rechte
w — World/Othersalle übrigen Benutzer des Systems

Mit ls -l werden die Rechte angezeigt, z. B. -rwxr-xr--: Owner darf alles (rwx), Gruppe darf lesen und ausführen (r-x), alle anderen dürfen nur lesen (r--).

Oktalnotation und chmod

RechtWert
r (read)4
w (write)2
x (execute)1

Die Werte einer Gruppe werden addiert: rwx = 7 (4+2+1), rw- = 6 (4+2), r-x = 5 (4+1). Drei Ziffern hintereinander (Owner-Group-Others) ergeben die Oktalnotation für chmod:

BefehlBedeutung
chmod 755 skript.shrwxr-xr-x — Owner alles, Gruppe/Andere lesen+ausführen
chmod 644 datei.txtrw-r--r-- — Owner lesen+schreiben, Rest nur lesen
chmod 600 privat.keyrw------- — nur Owner darf lesen/schreiben
chmod 777 datei.txtrwxrwxrwx — alle dürfen alles (fast immer ein Sicherheitsrisiko!)

Mit chown ben:users datei.txt wird der Eigentümer auf „ben“ und die Gruppe auf „users“ gesetzt. Die Option -R (z. B. chmod -R 755 /var/www/) wendet eine Änderung rekursiv auf einen ganzen Ordner samt Inhalt an.

BefehlZweck

Praxisbeispiel

Ein Techniker der TechNord GmbH richtet einen neuen Linux- Webserver ein:

  • Externe Backup-Platte einhängen: mount /dev/sdb1 /mnt/backup, danach mit ls -l /mnt/backup den Inhalt und die Rechte prüfen.
  • Deploy-Skript ausführbar machen: chmod 755 deploy.sh — Owner darf alles, alle anderen nur lesen und ausführen.
  • Web-Verzeichnis dem Webserver-Prozess übergeben: chown www-data:www-data /var/www/technord, damit der Webserver-Dienst korrekt darauf zugreifen kann.
  • Dauerhaftes Einhängen: Eintrag der Backup-Platte in /etc/fstab, damit sie bei jedem Neustart automatisch verfügbar ist.
  • Fehlersuche: grep "error" /var/log/nginx/error.log filtert gezielt Fehlermeldungen aus einer langen Logdatei.

Merksatz

Linux hat einen Verzeichnisbaum statt Laufwerksbuchstaben, Datenträger werden gemountet, /etc/fstab steuert das automatische Einhängen. Rechte gelten für Owner/Group/Others (ugw) je r/w/x — Oktalwerte r=4, w=2, x=1 addiert ergeben Werte wie 755, 644, 600. chmod 777 ist fast immer ein Sicherheitsrisiko. chown ändert den Eigentümer, grep durchsucht Dateien nach Textmustern.
