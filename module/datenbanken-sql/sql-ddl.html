SQL & DDL Kern AP2

Einstieg

Bevor TowerTech GmbH auch nur eine einzige Kundenakte digital ablegen kann, braucht sie einen Aktenschrank mit passenden Fächern: eines für Abteilungen, eines für Mitarbeitende. Erst wenn der Schrank mit seinen Fächern steht, ergibt es einen Sinn, Akten (Daten) hineinzulegen. Genau das ist die Aufgabe von SQL in seiner strukturgebenden Rolle: bevor irgendetwas gespeichert wird, muss erst einmal festgelegt werden, wie die Datenbank überhaupt aufgebaut ist.

Konzept

SQL (Structured Query Language, strukturierte Abfragesprache) ist die Standardsprache für relationale Datenbanken. Mit ihr werden Daten abgefragt, verändert, die Struktur der Datenbank definiert und Zugriffsrechte vergeben. SQL gliedert sich in drei Unterbereiche:

BereichZweckBefehle
DDL (Data Definition Language)Struktur der Datenbank festlegenCREATE, ALTER, DROP, TRUNCATE
DML (Data Manipulation Language)Daten lesen und verändernSELECT, INSERT, UPDATE, DELETE
DCL (Data Control Language)Zugriffsrechte steuernGRANT, REVOKE

Diese Einheit behandelt DDL im Detail; DML und DCL folgen in eigenen Einheiten.

Eine SQL-Anweisung (Query) endet mit einem Semikolon ;. Schlüsselwörter werden per Konvention großgeschrieben, SQL selbst ist nicht case-sensitiv – select und SELECT sind identisch.

SQL-Systeme im Vergleich

SQL ist standardisiert, aber jedes Datenbanksystem spricht einen eigenen Dialekt mit kleinen Abweichungen:

SystemStärkeTypischer Einsatz

Beispiel für Dialekt-Unterschiede: Eine automatisch hochzählende ID heißt in MySQL AUTO_INCREMENT, in PostgreSQL SERIAL, in SQLite AUTOINCREMENT. Der SQL-Editor in dieser Einheit läuft auf SQLite.

CREATE TABLE

Legt eine neue Tabelle mit ihren Spalten und Constraints an:

TabelleSpalten
abteilungabteilung_id (PK), name, buero_standort
mitarbeitermitarbeiter_id (PK), vollname, email (UNIQUE), jahresgehalt (DEFAULT 0), abteilung_id (FK → abteilung)

Wichtig: Zeigt eine Tabelle per Fremdschlüssel auf eine andere, muss die referenzierte Tabelle zuerst angelegt werden – abteilung vor mitarbeiter.

ALTER TABLE

Ändert eine bestehende Tabelle nachträglich – Spalten hinzufügen, umbenennen oder die Tabelle selbst umbenennen. Beispiel: TowerTech führt Homeoffice-Tage ein und muss die Tabelle mitarbeiter um eine Spalte erweitern, ohne vorhandene Daten zu verlieren.

DROP TABLE & TRUNCATE TABLE

BefehlWirkungDaten weg?Struktur weg?

DROP TABLE IF EXISTS … löscht eine Tabelle nur, wenn sie existiert – kein Fehler, falls nicht. SQLite kennt kein TRUNCATE; dort wird stattdessen DELETE FROM tabelle; verwendet.

Index

Ein Index beschleunigt Abfragen auf bestimmte Spalten (z. B. die häufige Suche nach einer E-Mail-Adresse), kostet dafür zusätzlichen Speicherplatz und verlangsamt INSERT/UPDATE/DELETE leicht, weil der Index mit aktualisiert werden muss. Primärschlüssel und UNIQUE-Spalten sind automatisch indiziert.

Praxisbeispiel

Der Chef von TowerTech GmbH gibt einer Auszubildenden vier Aufgaben für den SQL-Editor oben:

  • Aufgabe 1: Tabelle abteilung anlegen mit abteilung_id (INTEGER PRIMARY KEY), name (VARCHAR(100) NOT NULL), buero_standort (VARCHAR(100)).
  • Aufgabe 2: Tabelle mitarbeiter anlegen mit mitarbeiter_id (INTEGER PRIMARY KEY), vollname (VARCHAR(100) NOT NULL), email (VARCHAR(150) UNIQUE), jahresgehalt (DECIMAL(10,2) DEFAULT 0.00), abteilung_id (INTEGER, FOREIGN KEY auf abteilung.abteilung_id).
  • Aufgabe 3: Nachträglich per ALTER TABLE die Spalte homeoffice_tage (INTEGER, Standardwert 0) zur Tabelle mitarbeiter hinzufügen.
  • Aufgabe 4: Eine alte Testtabelle alte_testdaten entfernen – aber ohne Fehler, falls sie gar nicht existiert (DROP TABLE IF EXISTS).

Tipp: Mit PRAGMA table_info(mitarbeiter); lässt sich im Editor prüfen, welche Spalten eine Tabelle gerade hat.

Typische Prüfungsfrage: „Warum muss beim Anlegen von Tabellen mit Fremdschlüsselbeziehung eine bestimmte Reihenfolge eingehalten werden?" – Antwort: Der Fremdschlüssel muss auf einen bereits existierenden Primärschlüssel zeigen können, also muss die referenzierte Tabelle zuerst angelegt werden.

Merksatz

SQL gliedert sich in DDL (Struktur), DML (Daten) und DCL (Rechte). DDL-Befehle (CREATE, ALTER, DROP, TRUNCATE) legen die Tabellenstruktur fest und verändern sie – referenzierte Tabellen müssen vor verweisenden Tabellen angelegt werden. SQL ist standardisiert, aber jedes DBMS spricht einen eigenen Dialekt mit kleinen Abweichungen.
