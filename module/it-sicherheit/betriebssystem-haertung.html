Betriebssystem-Härtung Kern AP1 AP2

Einstieg

Bevor eine Familie in den Urlaub fährt, geht sie die Wohnung durch: alle Fenster zu, die selten genutzte Kellertür extra verriegeln, den Ersatzschlüssel beim Nachbarn nicht mehr sichtbar unter der Fußmatte liegen lassen. Nichts davon macht die Wohnung „sicherer im Alltag“ – es reduziert nur die Anzahl der Wege, über die jemand unbefugt hineinkommen könnte. Genau das ist Härtung bei einem Betriebssystem: jede unnötige Tür schließen, bevor sie zum Problem wird.

Konzept

Betriebssystem-Härtung (Hardening) bedeutet, ein System so zu konfigurieren, dass es möglichst wenige Angriffsflächen bietet. Leitprinzip ist die minimale Angriffsfläche (Attack Surface Reduction): Alles, was nicht gebraucht wird, ist ein potenzielles Risiko und wird deaktiviert oder entfernt.

Typische Härtungsmaßnahmen

MaßnahmeWarum
Unnötige Dienste/Prozesse deaktivierenJeder laufende Dienst ist ein möglicher Angriffspunkt, auch wenn er nie genutzt wird
Standardkonten umbenennen/deaktivierenKonten wie „Administrator“ oder „root“ sind Angreifern als Ziel bekannt und werden zuerst angegriffen
Standardpasswörter sofort ändernWerkseinstellungen sind öffentlich dokumentiert (Handbücher, Foren)
Nicht benötigte Software deinstallierenJede installierte Software kann eigene Schwachstellen mitbringen
Patch-Management: Updates zeitnah einspielenBekannte Sicherheitslücken werden sonst zum offenen Einfallstor
Nicht benötigte Netzwerk-Ports schließenJeder offene Port ist ein möglicher Angriffsvektor
Autostart-Programme prüfenSchadsoftware nistet sich gern im Autostart ein, um Neustarts zu überleben

Statt jede Maßnahme einzeln zu erfinden, orientieren sich Unternehmen häufig an fertigen Baseline-Konfigurationen wie den CIS Benchmarks (Center for Internet Security) – dokumentierten, geprüften Konfigurationsvorlagen für eine sichere Grundeinstellung verschiedener Betriebssysteme.

Server werden dabei in der Regel deutlich strenger gehärtet als Client-Rechner: Ein Server hat einen klar definierten Zweck (z. B. „nur Webserver“) und kann entsprechend fast alles Unnötige abschalten, während ein Arbeitsplatz-PC flexibler bleiben muss, weil Nutzer wechselnde Anwendungen brauchen.

Praxisbeispiel

Ein neuer Server wird bei der „Vogler IT-Dienste GmbH“ für einen Kunden aufgesetzt, der ausschließlich als Webserver dienen soll.

  • Der Administrator deaktiviert alle vorinstallierten Dienste, die nicht zum Webserver-Betrieb gehören (z. B. Dateifreigabe, Druckdienste).
  • Das Standardkonto „Administrator“ wird umbenannt und ein neues, individuelles Administrator-Konto mit langem Passwort und MFA angelegt.
  • Nur Port 443 (HTTPS) wird in der Firewall des Servers freigegeben, alle anderen Ports bleiben geschlossen – konsistent mit dem Prinzip der minimalen Angriffsfläche.
  • Ein automatisches Patch-Management-System spielt sicherheitsrelevante Updates innerhalb von 48 Stunden nach Veröffentlichung ein.
  • Die Konfiguration wird anschließend gegen einen CIS Benchmark für das eingesetzte Betriebssystem geprüft, um keine Standardeinstellung zu übersehen.

Typische Prüfungsfrage: „Nennen Sie drei Maßnahmen zur Betriebssystem-Härtung eines neuen Servers.“ – Antwort z. B.: unnötige Dienste deaktivieren, Standardkonten umbenennen, zeitnahes Patch-Management.

Merksatz

Härtung = minimale Angriffsfläche: alles Unnötige deaktivieren, entfernen oder schließen (Dienste, Konten, Ports, Software). Baseline-Vorlagen wie CIS Benchmarks liefern geprüfte Ausgangskonfigurationen. Server werden meist strenger gehärtet als Client-Rechner.
