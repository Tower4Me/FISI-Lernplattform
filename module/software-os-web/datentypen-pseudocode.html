Datentypen, Kontrollstrukturen & Pseudocode Kern AP1

Einstieg

Eine Variable ist wie eine beschriftete Kiste, in der über die Zeit unterschiedlicher Inhalt liegen kann — solange es zur angegebenen Kistenart passt. Kontrollstrukturen sind wie Verkehrsschilder auf einer Reise: manche Kreuzungen bedeuten „ist diese Straße gesperrt, nimm die andere“ (Verzweigung), andere „fahr diese Schleife, bis der Tank leer ist“ (Schleife). Und Pseudocode ist, als würdest du ein Rezept erst in eigenen Worten aufschreiben, bevor es ein stur-wörtlich denkender Kochroboter (die konkrete Programmiersprache) verstehen muss.

Konzept

Variablen und Datentypen

Eine Variable ist ein benannter Speicherplatz für einen Wert, der sich während der Programmlaufzeit ändern kann. Der Datentyp legt fest, welche Art von Wert gespeichert werden darf:

DatentypBeispielwert
Ganzzahl (Integer)42
Gleitkommazahl (Float/Double)3.14
Zeichenkette (String)"Hallo Welt"
Wahrheitswert (Boolean)true / false

Kontrollstrukturen

StrukturBedeutung
SequenzAnweisungen werden schlicht nacheinander ausgeführt — der Standardfall
Verzweigung (if/else)abhängig von einer Bedingung wird einer von mehreren möglichen Pfaden ausgeführt
Schleife (for/while)ein Anweisungsblock wird wiederholt ausgeführt, solange bzw. bis eine Bedingung erfüllt ist

Bei Schleifen wird zusätzlich unterschieden: Eine kopfgesteuerte Schleife (while) prüft die Bedingung vor jedem Durchlauf — trifft sie von Anfang an nicht zu, läuft die Schleife kein einziges Mal. Eine fußgesteuerte Schleife (do-while) prüft die Bedingung nach dem Durchlauf und läuft deshalb mindestens einmal, selbst wenn die Bedingung nie zutrifft.

Pseudocode

Pseudocode ist eine informelle, an natürlicher Sprache orientierte Notation zur Beschreibung eines Algorithmus — unabhängig von der Syntax einer konkreten Programmiersprache. Er dient dazu, einen Algorithmus zu planen und mit anderen zu besprechen, bevor er in einer bestimmten Sprache umgesetzt wird. Typische Schlüsselwörter: WENN … DANN … SONST, SOLANGE … TUE.

ALGORITHMUS GrößteZahl(a, b, c) WENN a >= b UND a >= c DANN ERGEBNIS = a SONST WENN b >= c DANN ERGEBNIS = b SONST ERGEBNIS = c ENDE WENN RÜCKGABE ERGEBNIS

Praxisbeispiel

Ein Auszubildender der TechNord GmbH soll einen Algorithmus entwickeln, der prüft, ob ein Kunde Anspruch auf Mengenrabatt hat:

FÜR JEDE Position IN Bestellliste WENN Position.Menge >= 100 DANN Position.Rabatt = 10% SONST Position.Rabatt = 0% ENDE WENN ENDE FÜR

Der Azubi skizziert den Algorithmus zunächst in Pseudocode — so kann der Ausbilder die Logik prüfen und Rückmeldung geben, ohne dass beide dieselbe Programmiersprache im Detail beherrschen müssen. Erst danach wird der Algorithmus in eine konkrete Sprache (hier Python) übersetzt. Die FÜR JEDE-Schleife durchläuft dabei jede Position der Bestellliste, die WENN-Verzweigung entscheidet pro Position über den Rabattsatz.

Merksatz

Variable = benannter, veränderlicher Speicherplatz; Datentyp legt die erlaubte Art des Werts fest (Integer, Float, String, Boolean). Kontrollstrukturen: Sequenz, Verzweigung (if/else), Schleife (kopf- vs. fußgesteuert). Pseudocode beschreibt einen Algorithmus sprachunabhängig in natürlicher Sprache, bevor er in einer konkreten Programmiersprache umgesetzt wird.
