Programmierparadigmen & Sprachhöhe Kern AP1

Einstieg

Prozedurale Programmierung ist wie ein Kochrezept mit nummerierten Schritten, die von oben nach unten abgearbeitet werden. Objektorientierte Programmierung ist wie ein Unternehmen, das in Abteilungen (Objekte) organisiert ist — jede Abteilung verwaltet ihre eigenen Daten und Aufgaben und kommuniziert über klar definierte Schnittstellen mit den anderen. Funktionale Programmierung ist wie ein Fließband, auf dem jede Station nur das durchlaufende Werkstück verändert, ohne je auf ein gemeinsames Lager zuzugreifen.

Konzept

Ein Programmierparadigma ist ein grundlegender Stil, wie ein Problem in Code strukturiert wird.

Die drei wichtigsten Paradigmen

ParadigmaGrundprinzipBeispielsprachen
Prozedural das Programm ist eine Folge von Anweisungen und Prozeduren (Funktionen), die nacheinander ausgeführt werden und auf gemeinsame Daten zugreifen C, klassisches Pascal
Objektorientiert (OOP) Daten und die dazugehörigen Funktionen werden in Objekten bzw. Klassen gebündelt (Kapselung); weitere Kernkonzepte: Vererbung (eine Klasse übernimmt Eigenschaften einer anderen) und Polymorphie (gleicher Aufruf, unterschiedliches Verhalten je nach Objekt) Java, C#, Python
Funktional das Programm besteht aus der Verkettung von (möglichst zustandslosen) Funktionen, ohne einen gemeinsamen veränderlichen Zustand zu verändern Haskell; funktionale Konzepte auch in JavaScript/Python nutzbar

Sprachhöhe: Maschinensprache bis Hochsprache

EbeneMerkmale
Maschinensprache reine Binärbefehle, direkt vom Prozessor ausführbar, prozessorspezifisch, für Menschen praktisch nicht lesbar
Assemblersprache mnemonische Abkürzungen für Maschinenbefehle (z. B. MOV, ADD), noch sehr hardwarenah, aber für Menschen lesbarer
Hochsprache abstrahiert von der konkreten Hardware, deutlich menschenlesbarer und portabler zwischen unterschiedlichen Prozessoren (z. B. C, Java, Python)

Je höher die Sprachebene, desto weiter weg vom konkreten Prozessor, aber desto leichter zu schreiben, zu lesen und zu warten — dafür übernimmt ein Compiler oder Interpreter die Übersetzung in Maschinensprache (siehe Einheit Compiler, Interpreter & Linker).

Praxisbeispiel

Die TechNord GmbH entwickelt zwei unterschiedliche Projekte:

  • Automatisierungsskripte für die Serverwartung: werden prozedural als einfache Python-Skripte geschrieben — Schritt für Schritt abgearbeitet, ohne Bedarf an einer komplexen Objektstruktur.
  • Kundenverwaltungssoftware: wird objektorientiert in Java entwickelt. Reale Objekte wie Kunde, Bestellung und Rechnung lassen sich gut als Klassen mit eigenen Daten und Methoden abbilden. Durch Vererbung erben die Klassen „Privatkunde“ und „Geschäftskunde“ gemeinsame Eigenschaften von einer Basisklasse „Kunde“, statt Code doppelt zu schreiben.

Merksatz

Prozedural = Abfolge von Prozeduren auf gemeinsamen Daten. Objektorientiert = Daten und Funktionen in Objekten/Klassen gebündelt (Kapselung, Vererbung, Polymorphie). Funktional = Verkettung zustandsloser Funktionen. Sprachhöhe: Maschinensprache (prozessorspezifisch) → Assembler (mnemonisch) → Hochsprache (abstrakt, portabel, menschenlesbar).
