Dateisysteme-Grundlagen Kern AP1 AP2

Einstieg

Der Inhalt eines Briefes verstehen nur Absender und Empfänger — die Post transportiert ihn nur, ohne den Text zu lesen. Genauso verwaltet ein Betriebssystem Dateien: Es weiß, wo eine Datei liegt und wie sie heißt, versteht aber nicht, was in ihr steht — das interpretiert erst das öffnende Programm. Alles, was außen auf dem Umschlag steht (Absender, Datum, Größe), sind die Metadaten. Und damit die Post überhaupt weiß, in welchem Fach ein Brief physisch liegt, führt sie ein internes Register — bei Linux heißt dieses Register Inode, bei Windows/NTFS Master File Table.

Konzept

Was ist eine Datei?

Auf unterster Ebene ist eine Datei nichts anderes als eine benannte, geordnete Folge von Bytes auf einem Speichermedium. Das Betriebssystem weiß nicht, was inhaltlich in der Datei steckt — es verwaltet nur, wo sie liegt und wie sie heißt. Die Dateiendung (z. B. .jpg) ist dabei nur ein Hinweis für das Betriebssystem, welches Programm die Datei öffnen soll — sie kann umbenannt werden, ohne den Inhalt zu verändern. Der tatsächliche Dateityp steckt in den ersten Bytes der Datei selbst (siehe Einheit Zahlensysteme, Hex-Viewer/Magic Bytes).

Metadaten

Metadaten (Daten über eine Datei, nicht der Inhalt selbst) werden vom Dateisystem getrennt vom eigentlichen Dateiinhalt gespeichert:

MetadatumBedeutung
Namemenschenlesbarer Dateiname inkl. Endung — wird vom Dateisystem verwaltet, nicht in der Datei selbst gespeichert
Größetatsächliche Dateigröße in Bytes (im Unterschied zur „Größe auf Datenträger“, siehe Einheit Dateisysteme-Vergleich)
ZeitstempelErstellung, letzte Änderung, letzter Zugriff — forensisch relevant
Berechtigungenwer lesen/schreiben/ausführen darf — unter Linux rwx, unter Windows ACL (siehe die beiden vorherigen Einheiten)
SpeicherortZeiger auf die tatsächlichen Datenblöcke auf dem Datenträger (Inode bzw. MFT-Eintrag)
Attributezusätzliche Flags wie „versteckt“ oder „System“

Inode (Linux)

Unter Linux verwaltet das Dateisystem jede Datei über einen Inode (Index Node). Der Inode enthält alle Metadaten außer dem Dateinamen — der Dateiname ist nur ein Verweis auf den Inode.

BefehlZweck
ls -i datei.txtzeigt die Inode-Nummer einer Datei
stat datei.txtzeigt alle Metadaten: Größe, Blöcke, Zugriffsrechte, Inode, Zeitstempel

Wird eine Datei „gelöscht“, wird meist nur der Inode-Eintrag als frei markiert — die eigentlichen Daten bleiben auf dem Datenträger, bis sie überschrieben werden. Genau deshalb lassen sich gelöschte Dateien mit Forensik-Tools oft wiederherstellen.

MFT — Master File Table (Windows/NTFS)

Windows NTFS nutzt statt Inodes die MFT (Master File Table). Jede Datei besitzt einen MFT-Eintrag mit allen Metadaten; bei sehr kleinen Dateien (unter ca. 700 Byte) wird der Inhalt sogar direkt im MFT-Eintrag gespeichert, ohne eigenen Datenblock.

Was macht ein Dateisystem allgemein?

Ein Dateisystem löst vier Grundprobleme:

  • Organisation — hierarchische Struktur aus Verzeichnissen und Dateien
  • Adressierung — welche Blöcke auf dem Datenträger zu welcher Datei gehören
  • Freispeicherverwaltung — welche Blöcke belegt und welche frei sind
  • Metadatenverwaltung — Name, Größe, Zeitstempel, Berechtigungen

Praxisbeispiel

Ein Mitarbeiter der TechNord GmbH löscht versehentlich eine wichtige Projektdatei von seinem Linux-Arbeitsplatz:

  • Erklärung durch den Support: Beim Löschen wurde nur der Inode-Eintrag als frei markiert — die eigentlichen Datenbytes liegen wahrscheinlich noch unverändert auf dem Datenträger.
  • Sofortmaßnahme: Der Rechner wird nicht weiter genutzt, damit keine neuen Daten die alten Blöcke überschreiben.
  • Wiederherstellung: Ein Datenrettungstool durchsucht den Datenträger nach noch nicht überschriebenen Datenblöcken und stellt die Datei erfolgreich wieder her.
  • Zur Dokumentation zeigt der Techniker mit stat anschließend alle Metadaten der wiederhergestellten Datei — inklusive der ursprünglichen Zeitstempel.

Merksatz

Eine Datei ist eine Byte-Folge, deren Inhalt nur das öffnende Programm versteht — die Dateiendung ist nur ein Hinweis fürs OS. Metadaten (Name, Größe, Zeitstempel, Berechtigungen, Speicherort) werden getrennt vom Inhalt gespeichert. Linux nutzt dafür den Inode (Name ist nur ein Verweis darauf), Windows/NTFS die MFT. Löschen markiert meist nur den Metadaten-Eintrag als frei — die Daten bleiben zunächst erhalten.
