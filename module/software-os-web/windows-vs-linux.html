Windows vs. Linux/Unix & Domäne Kern AP1 AP2

Einstieg

Windows ist wie eine möblierte Mietwohnung: komfortabel eingerichtet, mit breiter Geräte- und Software-Unterstützung, aber man zahlt Miete und die Einrichtung ist vorgegeben. Linux ist wie ein unmöbliertes Loft: kostenlos in der Grundausstattung und komplett nach eigenen Vorstellungen anpassbar, verlangt dafür aber mehr Eigeninitiative beim Einrichten. Und der Beitritt zu einer Domäne ist wie der Umzug in eine verwaltete Wohnanlage mit zentraler Hausverwaltung (Domain Controller), die für alle Bewohner gemeinsam Schlüssel und Zutrittsregeln ausgibt — statt dass jede Wohnung ihr eigenes Schloss unabhängig verwaltet (Arbeitsgruppe).

Konzept

Windows vs. Linux/Unix

WindowsLinux/Unix
Quellcode proprietär (Microsoft), nicht einsehbar meist Open Source (GNU/Linux-Distributionen wie Ubuntu, Debian, RHEL) — Kernel und viele Tools offen einsehbar
Lizenzkosten kommerzielle Lizenzgebühr meist kostenlos, kommerzieller Support optional zusätzlich buchbar (z. B. Red Hat)
Typischer Einsatz dominierend auf Desktop-Arbeitsplätzen dominierend auf Servern, in Cloud-Infrastruktur und eingebetteten Systemen
Bedienung traditionell GUI-zentriert, PowerShell für Automatisierung traditionell kommandozeilenorientiert (Shell/Bash), viele Server laufen komplett ohne grafische Oberfläche (headless)

Auch macOS zählt technisch zu den Unix-artigen Systemen (BSD- Abstammung), spielt aber im professionellen Server-Umfeld eine deutlich kleinere Rolle als Linux.

Arbeitsgruppe vs. Domäne

In einer Arbeitsgruppe verwaltet jeder Rechner seine Benutzerkonten und Zugriffsrechte vollständig lokal und unabhängig von den anderen Rechnern — praktikabel nur für sehr kleine Netzwerke. In einer Domäne übernimmt ein zentraler Domain Controller (bei Windows: Active Directory) die Benutzer- und Rechteverwaltung für alle Mitgliedsrechner zentral.

Vorteil der DomäneBedeutung
Zentrale Anmeldungein Benutzerkonto meldet sich an jedem Domänenrechner mit denselben Zugangsdaten an
Gruppenrichtlinien (GPO)zentral definierte Sicherheits- und Konfigurationsvorgaben gelten automatisch für alle betroffenen Rechner (siehe Modul IT-Sicherheit, Berechtigungskonzepte)
Einfachere VerwaltungRechte, Passwortrichtlinien und Software lassen sich zentral statt pro Einzelgerät pflegen

Auch Linux-Rechner lassen sich einer bestehenden Active-Directory-Domäne hinzufügen (z. B. über Samba/Winbind oder SSSD) — so können gemischte Windows-/Linux-Umgebungen zentral mit denselben Zugangsdaten verwaltet werden.

Praxisbeispiel

Die TechNord GmbH betreibt eine gemischte Umgebung:

  • 50 Windows-Arbeitsplätze werden in eine Active-Directory-Domäne eingebunden — jeder Mitarbeiter meldet sich an jedem Rechner mit demselben Konto an, Passwortrichtlinien und Softwareverteilung laufen zentral über Gruppenrichtlinien.
  • Mehrere Linux-Server laufen headless (ohne grafische Oberfläche) für Webservices und Datenbanken — das spart Ressourcen und läuft im Dauerbetrieb besonders stabil.
  • Ein neuer Linux-Entwickler-Arbeitsplatz wird zusätzlich per SSSD in dieselbe Active-Directory-Domäne eingebunden, damit sich der Entwickler mit denselben Zugangsdaten anmelden kann wie an den Windows-Rechnern.

Merksatz

Windows: proprietär, GUI-zentriert, Desktop-dominant. Linux/Unix: meist Open Source, kommandozeilenorientiert, Server-dominant. Arbeitsgruppe = dezentrale, lokale Verwaltung pro Rechner. Domäne = zentraler Domain Controller (Active Directory) für Anmeldung, Gruppenrichtlinien und Rechteverwaltung — auch Linux-Rechner können einer AD-Domäne beitreten.
