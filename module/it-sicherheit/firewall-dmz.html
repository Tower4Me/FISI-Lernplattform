Firewall & DMZ Kern AP1 AP2

Einstieg

Ein Firmengebäude hat oft einen Empfangsbereich, den auch externe Besucher betreten dürfen, ohne gleich Zutritt zu den Büros im Inneren zu bekommen. Zwischen Empfang und Büros liegt eine weitere verschlossene Tür. Selbst wenn im Empfangsbereich etwas schiefgeht, bleibt der eigentliche Kernbereich geschützt. Genauso funktioniert eine Firewall mit einer DMZ im Netzwerk.

Konzept

Eine Firewall kontrolliert den Datenverkehr zwischen Netzen anhand von Regeln. Sie ist die wichtigste Grenze zwischen „innen“ (vertrauenswürdig) und „außen“ (Internet). Es gibt verschiedene Ausbaustufen:

TypArbeitet aufErkennt
PaketfilterSchicht 3/4 (IP, Port)Quelle, Ziel, Port, Protokoll
Stateful Inspection (SPI)Schicht 3/4 + VerbindungszustandGehört das Paket zu einer bestehenden Verbindung?
Application Firewall / NGFW / WAFbis Schicht 7Anwendung, Inhalt, Deep Packet Inspection

Eine Next-Generation-Firewall (NGFW) macht Stateful Inspection, kann aber zusätzlich Application Control und Deep Packet Inspection bis Schicht 7 – eine reine Paketfilter-Regel („Port 443 erlauben“) ist nur die unterste Ausbaustufe.

Die DMZ

Eine DMZ (Demilitarisierte Zone) ist ein Puffernetz zwischen Internet und internem LAN. Server, die von außen erreichbar sein müssen (Webserver, Mailserver), stehen hier. Wird ein DMZ-Server gehackt, hat der Angreifer noch keinen Zugriff auf das interne Netz – dazwischen liegt eine zweite Firewall-Grenze.

Praxisbeispiel

Die „Baumann IT-Systemhaus GmbH“ plant das Netzwerk für einen Kunden mit eigenem Webshop und internem Fileserver:

  • Der Webshop-Server muss von jedem Kunden im Internet erreichbar sein → er kommt in die DMZ, nicht ins interne LAN.
  • Firewall 1 (zwischen Internet und DMZ) lässt nur HTTPS-Verkehr (Port 443) zum Webshop-Server durch.
  • Firewall 2 (zwischen DMZ und internem LAN) lässt nur die eine Verbindung durch, die der Webshop-Server für Bestelldaten zur internen Datenbank benötigt – sonst nichts.
  • Ein Angreifer, der eine Schwachstelle im Webshop ausnutzt, landet damit nur in der DMZ. Um an den internen Fileserver mit den Kundendaten zu gelangen, müsste er zusätzlich Firewall 2 überwinden.
  • Eine reine Paketfilter-Regel würde nur Quelle/Ziel/Port prüfen; die eingesetzte NGFW erkennt zusätzlich, ob der Datenverkehr zur Anwendung „Webshop“ überhaupt plausibel aussieht (Deep Packet Inspection).

Merksatz

Firewall-Ausbaustufen: Paketfilter (L3/L4) → Stateful Inspection (+ Verbindungszustand) → NGFW/WAF (bis L7). Die DMZ ist ein Puffernetz zwischen zwei Firewalls für von außen erreichbare Server – ein kompromittierter DMZ-Server öffnet nicht automatisch das interne LAN.
