HTML/CSS-Grundlagen & Impressumspflicht Randwissen AP1

Einstieg

HTML ist wie der Rohbau eines Hauses: Wände, Räume, Türen — die reine Struktur und der Inhalt. CSS ist die Inneneinrichtung: Farben, Möbel, wie alles angeordnet ist. Responsive Design ist ein Haus, das sich automatisch neu einrichtet, je nachdem, ob man es als kleines Puppenhaus-Modell oder als große Villa betrachtet — derselbe Inhalt, aber ein anderes Layout je nach Bildschirmgröße. Und die Impressumspflicht ist wie ein gesetzlich vorgeschriebenes Namensschild am Ladeneingang, das verrät, wer tatsächlich hinter dem Geschäft steckt.

Konzept

HTML — Struktur der Webseite

HTML (HyperText Markup Language) beschreibt die Struktur und den Inhalt einer Webseite über Elemente (Tags) wie <h1> (Überschrift), <p> (Absatz), <a> (Link) oder <img> (Bild). Der Browser interpretiert diese Struktur und stellt sie entsprechend dar.

CSS — Aussehen der Webseite

CSS (Cascading Style Sheets) beschreibt getrennt vom HTML-Inhalt, wie die Seite aussieht: Farben, Abstände, Schriftarten, Layout. Diese Trennung von Struktur (HTML) und Design (CSS) erlaubt es, dieselbe Struktur mit unterschiedlichem Design darzustellen, ohne den Inhalt anfassen zu müssen.

Responsive Design

Responsive Design sorgt dafür, dass sich eine Webseite automatisch an unterschiedliche Bildschirmgrößen anpasst — Smartphone, Tablet, Desktop zeigen dieselben Inhalte in einem jeweils passenden Layout. Technisch umgesetzt wird das u. a. über:

  • Media Queries — CSS-Regeln, die abhängig von der Bildschirmbreite unterschiedliche Stile anwenden (z. B. Navigationsleiste wird bei schmalen Bildschirmen zu einem zusammenklappbaren Menü)
  • Flexible Layouts (z. B. Flexbox oder Grid) statt starrer Pixel-Positionierung
  • Relative Einheiten statt fester Pixelwerte, damit Elemente mitskalieren

Impressumspflicht

In Deutschland besteht für praktisch jede geschäftsmäßige Website eine gesetzliche Impressumspflicht. Pflichtangaben umfassen u. a.:

  • Name und Anschrift des Betreibers
  • eine direkte Kontaktmöglichkeit (z. B. E-Mail-Adresse)
  • bei Kapitalgesellschaften: Handelsregistereintrag und Registernummer
  • gegebenenfalls Umsatzsteuer-Identifikationsnummer

Ein fehlendes oder unvollständiges Impressum kann rechtlich abgemahnt werden — es ist deshalb keine Kür, sondern verpflichtender Bestandteil jeder geschäftlichen Website.

Praxisbeispiel

Die TechNord GmbH überarbeitet ihre Firmenwebsite:

  • HTML-Struktur (Überschriften, Texte, Navigation) bleibt inhaltlich unverändert.
  • Neues CSS mit Media Queries sorgt dafür, dass die volle Desktop-Navigationsleiste auf dem Smartphone zu einem kompakten Hamburger-Menü zusammenklappt — ohne dass am HTML-Inhalt etwas geändert werden musste.
  • Rechtssicheres Impressum: Die alte Website enthielt weder die Handelsregisternummer noch die Umsatzsteuer-Identifikationsnummer — beides wird nun ergänzt, um eine mögliche Abmahnung wegen unvollständiger Pflichtangaben zu vermeiden.

Merksatz

HTML beschreibt Struktur/Inhalt, CSS beschreibt das Aussehen — beide sind bewusst getrennt. Responsive Design passt das Layout über Media Queries, flexible Layouts und relative Einheiten an die Bildschirmgröße an. Jede geschäftsmäßige Website in Deutschland braucht ein vollständiges Impressum (Name, Anschrift, Kontakt, ggf. Handelsregister/USt-IdNr.) — sonst drohen Abmahnungen.
